## 📊 关键组件职责

### 1. **Telegram Extension**
- 接收 Telegram Webhook
- 解析和格式化消息
- 调用 Telegram Bot API

### 2. **Gateway**
- 消息路由和分发
- 权限验证
- 事件广播
- Node 管理

### 3. **Agent**
- 消息理解和意图识别
- 上下文管理
- LLM 调用
- 工具执行决策

### 4. **Node**
- 执行设备操作
- 访问本地资源
- 返回执行结果

---

## ⏱️ 时间估算

| 步骤 | 耗时 |
|------|------|
| Telegram → Extension | ~100ms |
| Extension → Gateway | ~50ms |
| Gateway → Agent | ~50ms |
| Agent LLM 调用 | ~1-2s |
| Agent → Node | ~100ms |
| Node 执行截图 | ~500ms |
| Node → Agent | ~100ms |
| Agent 生成响应 | ~500ms |
| Agent → Gateway | ~50ms |
| Gateway → Extension | ~50ms |
| Extension → Telegram | ~200ms |
| **总计** | **~3-4s** |

## 🔐 安全机制

### 1. **认证**
- Telegram Bot Token
- Gateway Token
- Node Device ID

### 2. **授权**
- 用户权限验证
- Node 能力声明
- 工具调用权限

### 3. **隐私**
- 截图文件本地存储
- 不上传到云端
- 自动清理临时文件
